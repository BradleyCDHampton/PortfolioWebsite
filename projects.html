Overview

A prototype for an Android Mobile application, which matches and connects Gamemasters and Players together based on their preferences for a TTRPG Campaign.

Links

  • Github - full source code
  • Project Website - additional documentation and resources

Technologies Used

  • Android Studio
  • Java
  • Python (Flask)
  • MySQL
  • Docker
  • Jira

Motivation

Finding and organizing groups for TTRPGs poses a logistical challenge. A campaign requires consistent commitment and coordination from all group members. Players and GMs must all align on:

  • what game they will play
  • where they will meet
  • when sessions will take place
  • how difficult the Campaign will be
  • what roles each character will play as

A mismatch in player expectations or preferences causes friction and could terminate a Campaign. GMs must carefully consider whether each potential Player fits the Campaign they intend to run. TTRPG players traditionally find groups to play with in-person or using online forums. These methods are inefficient and time-consuming.

Achievements

  • Organized a Jira taskboard to divide work between Sprints and team members following Agile methodology
  • Connected the front-end and back-end via Flask routes within a Docker containerized environment
  • Created entity-relationship diagrams to derive the MySQL database schema used throughout the Project
  • Designed and implemented a text-based chatroom to enable group members to communicate after matching

Overview

Generates a solution for a given Sokomind puzzles with 6 different search algorithms:
  • Depth-First Search
  • Breadth-First Search
  • Greedy Search w/ a Manhattan Heuristic
  • Greedy Search w/ a Custom (documented) Heuristic
  • A* Search w/ a Manhattan Heuristic
  • A* Search w/ a Custom (documented) Heuristic

Links

  • Download - full source code
  • Report - additional documentation and analysis

Technologies Used

  • Python (argparse, numpy)

Motivation

An investigation of the memory cost, runtime cost, and scalability of 6 different search algorithms.

Achievements

  • Implemented search algorithms and heuristics as high-order functions, enabling all code to be shared when any algorithm is used.
  • Reduced memory usage by 91% and runtime by 45% by designing a custom heuristic tailored to the problem, compared to a Manhattan heuristic.
  • Leveraged the Python argparse library to enable runtime options to be selected with command-line arguments.

Overview

A game of Reversi where input is recieved via a Graphical UI, and the computer-controlled opponent's moves are determined by configuring how the computer evaluates the game-state and how far it should look-ahead.

Links

  • Download - full source code
  • Report - additional documentation and analysis

Technologies Used

  • Python (tkinter, argparse)

Motivation

  • To create an AI opponent, which is able to defeat an intermediate-level player in Reversi, while making its in roughly less than 30 seconds per move.
  • Investigate the relative performances of min-max search and alpha-beta pruning when applied to different heuristic functions.

Achievements

  • Explored multithreaded control flow through creating a graphical UI using Python's Tkinter package.
  • Applied k-fold alpha-beta lookahead creating computer-controlled opponents at an advanced skill level.
  • Implemented 3 distinct heuristic functions (documented in the report).
  • Created command-line arguments enabling the algorithm, heuristic, and depth of look-ahead to be selected when the program is run.
